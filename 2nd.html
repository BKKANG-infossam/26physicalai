실물 ESP32 PLUS 보드를 이용한 디지털/아날로그 제어 종합 실습 안내

1. 우리가 사용하는 고성능 하드웨어 스펙

우리가 실습에 활용하는 Keyestudio ESP32 PLUS 보드는 기본 아두이노 우노(UNO) 보드와 핀 배열은 비슷하지만 내부 구조와 연산 능력은 압도적인 차이를 보입니다. 향후 7~8차시에서 다룰 기기 독자형 인공지능 알고리즘(TinyML) 연산을 수행할 수 있도록 무선 와이파이, 블루투스 모듈 및 초고속 프로세서가 탑재되어 있습니다.

⚠️ 전압 제한 필수 경고!

조이스틱의 전원선(VCC)은 보드의 5V 단자가 아닌 반드시 3.3V 단자에 연결해야 합니다. 5V에 직접 전원을 투입할 경우, 아날로그 신호를 감지하는 ESP32 보드의 입력 한계 전압(3.3V)을 초과하여 조금만 조이스틱을 밀어도 전압 포화(Clipping) 현상이 발생합니다. 둥근 궤적 데이터를 정상적으로 얻기 위한 필수 공정입니다.

🕹️ 조이스틱 모듈 가변저항(VR)과 스위치(SW) 원리

조이스틱 변수명에 들어가는 VRx와 VRy의 VR은 Variable Resistor(가변저항)의 약자입니다. 조이스틱은 한마디로 좌우 방향용 가변저항 1개와 상하 방향용 가변저항 1개가 격자 형태로 결합된 복합 센서입니다. 막대를 조작하면 내부의 도선 접점이 회전하면서 저항값이 변화하고, 이 변화된 아날로그 전압의 크기를 보드가 실시간으로 감지합니다.

조이스틱의 수직 누름 스위치(SW)는 디지털 값을 전달하며, 아두이노 보드 내장 풀업 저항(INPUT_PULLUP) 설정을 이용해 누르면 0 (LOW), 손을 떼면 1 (HIGH)로 출력하는 역논리 방식으로 신호를 이해합니다.

모듈 인쇄 핀 ESP32 Plus 보드 소켓 위치 지정 변수명 추천 점퍼선 색상

모듈 인쇄 핀 ESP32 Plus 보드 소켓 위치 지정 변수명 추천 점퍼선 색상

3. [실습 1] 조이스틱 버튼을 이용한 디지털 입출력

디지털 신호는 오직 0과 1이라는 두 가지 상반된 상태만을 가집니다. 조이스틱 내부 스위치를 딸깍 누르면 LOW (0) 신호가 발생하고, 손을 떼면 내부 풀업 저항에 의해 HIGH (1) 신호가 유지됩니다. 누르면 빨간색 불이, 안 누르면 초록색 불이 켜지도록 코드를 한 줄씩 쳐 봅시다.

Digital_IO_Practice.ino 🔒 복사 및 선택 불가능 (직접 타이핑하세요) 
// 단축형 변수 설계로 코딩 오타 최소화
int RLED = 13; // 빨간색 LED 핀
int GLED = 14; // 초록색 LED 핀
int SW = 26;   // 조이스틱 버튼 스위치 핀

void setup() {
  pinMode(RLED, OUTPUT);
  pinMode(GLED, OUTPUT);
  // 내부 풀업 저항 모드로 설정하여 전기적 플로팅 노이즈 제거
  pinMode(SW, INPUT_PULLUP); 
}

void loop() {
  // 버튼의 신호 상태를 변수 st에 저장
  int st = digitalRead(SW); 
  
  if (st == 0) { // 버튼을 누른 경우 (LOW == 0)
    digitalWrite(RLED, 1); // 빨간색 ON
    digitalWrite(GLED, 0); // 초록색 OFF
  } else {       // 버튼을 누르지 않은 평상시
    digitalWrite(RLED, 0); // 빨간색 OFF
    digitalWrite(GLED, 1); // 초록색 ON
  }
}

4. [실습 2] 조이스틱 X, Y 변환값 시리얼 관측

아날로그 신호는 0과 1 사이의 무수히 연속적인 값의 변화를 뜻합니다. ESP32 Plus 보드는 12비트 해상도의 아날로그-디지털 변환기(ADC)를 사용하여 가변저항 전압의 흔들림을 0 ~ 4095 사이의 정밀한 숫자로 변환해 줍니다.

💡 아날로그 핀 선언 꿀팁

아두이노 컴파일 엔진은 analogRead() 함수가 호출되는 즉시 해당 핀의 제어권을 아날로그 회로로 강제 스위칭합니다. 따라서 setup() 함수 안에 아날로그 핀에 대한 pinMode(핀, INPUT) 선언을 과감히 생략하여 코드를 획기적으로 단축할 수 있습니다.

Analog_Monitor.ino 🔒 복사 및 선택 불가능 (직접 타이핑하세요) 
int X = 36; // 조이스틱 X축 아날로그 입력 핀
int Y = 39; // 조이스틱 Y축 아날로그 입력 핀

void setup() {
  Serial.begin(115200); // 통신 고속도로(115200) 개방
}

void loop() {
  int vx = analogRead(X); // X축 현재 좌표 읽기 (0~4095)
  int vy = analogRead(Y); // Y축 현재 좌표 읽기 (0~4095)
  
  // 시리얼 모니터 및 시리얼 플로터 가독성을 위한 문자열 가공 출력
  Serial.print(vx);
  Serial.print(" , ");
  Serial.println(vy);
  
  delay(100); // 0.1초마다 모니터링창에 업데이트
}

5. [실습 3] 조이스틱 방향에 따른 신호등 자동제어

실습 1의 디지털 출력 세팅과 실습 2의 아날로그 방향 판단 기준 조건을 결합합니다. 조이스틱을 가만히 두었을 때의 전압 중심값은 대략 2000 부근입니다. 왼쪽 끝으로 밀면 500 미만, 오른쪽 끝으로 밀면 3500 초과가 되는 특성을 조건문(if - else if)에 연동해 봅시다.

Joystick_Traffic_Control.ino 🔒 복사 및 선택 불가능 (직접 타이핑하세요) 
int X = 36; 
int Y = 39; 
int RLED = 13; // 신호등 빨강 LED 추가
int YLED = 12; // 신호등 노랑 LED 추가
int GLED = 14; // 신호등 초록 LED 추가

void setup() {
  Serial.begin(115200); 
  pinMode(RLED, OUTPUT); // 출력으로 개방
  pinMode(YLED, OUTPUT);
  pinMode(GLED, OUTPUT);
}

void loop() {
  int vx = analogRead(X); 
  int vy = analogRead(Y); 

  // 새로운 움직임을 감지하기 위해 일단 모든 LED 소등
  digitalWrite(RLED, 0);
  digitalWrite(YLED, 0);
  digitalWrite(GLED, 0);

  // 좌/우/상 세밀 방향 제어 (대각선 간섭 차단을 위한 다중 제어문 구조)
  if (vx < 500) {
    digitalWrite(YLED, 1); // 왼쪽으로 끝까지 밀면 노란불 ON
  } 
  else if (vx > 3500) {
    digitalWrite(GLED, 1); // 오른쪽으로 끝까지 밀면 초록불 ON
  } 
  else if (vy > 3500) {
    digitalWrite(RLED, 1); // 위쪽으로 끝까지 밀면 빨간불 ON
  }

  Serial.print(vx);
  Serial.print(" , ");
  Serial.println(vy);
  
  delay(20); // 부드러운 반응 조작을 위한 통신 딜레이 축소
}

⭐ 6. [최종 종합 과제] 버튼 우선순위 복합 알고리즘

지금까지 배운 디지털 입력(SW), 아날로그 입력(X, Y), 디지털 출력(LED 3종) 제어를 하나의 파일로 융합하고, 조건 우선순위를 부여하는 고난도 알고리즘 설계 문제입니다.

📋 도전과제 로직 및 시나리오 조건
  1. [1순위 대장 조건] 조이스틱 버튼(SW)을 누르면, 조이스틱 막대가 어디로 쏠려 있든지 상관없이 무조건 빨간색 LED가 켜집니다.
  2. [2순위 조건] 조이스틱 버튼을 누르지 않은 안전한 상태에서만 아래의 방향 제어를 판단합니다.
    • 조이스틱을 왼쪽으로 밀면 노란색 LED가 점등합니다.
    • 조이스틱을 오른쪽으로 밀면 초록색 LED가 점등합니다.
  3. [평상시 조건] 버튼도 안 누르고 조이스틱도 중간에 가만히 둔 경우 모든 LED가 꺼져야 합니다.

🧠 우선순위 제어 힌트 및 코드 설계

컴퓨터가 실행하는 if - else if 구문은 가장 위에 올라온 조건의 참 거짓을 먼저 판독하고, 만족하는 즉시 하단 조건문 영역 전체를 무시(스킵)합니다. 이 특성을 이용해 가장 강력한 방해 요인인 버튼 상태(st) 판독 조건을 가장 상위 if 문으로 선언해야 올바른 알고리즘이 성립됩니다.

Joystick_Priority_System.ino 🔒 복사 및 선택 불가능 (직접 타이핑하세요) 
int X = 36; 
int Y = 39; 
int SW = 26; 
int RLED = 13; 
int YLED = 12; 
int GLED = 14; 

void setup() {
  Serial.begin(115200); 
  pinMode(RLED, OUTPUT); 
  pinMode(YLED, OUTPUT); 
  pinMode(GLED, OUTPUT); 
  pinMode(SW, INPUT_PULLUP); // 디지털 입력 내부 풀업 필수선언
}

void loop() {
  int vx = analogRead(X); 
  int vy = analogRead(Y); 
  int st = digitalRead(SW); // 버튼 상태 파악 (안누르면 1, 누르면 0)

  // 버퍼 초기화를 위해 모든 불 끄기
  digitalWrite(RLED, 0);
  digitalWrite(YLED, 0);
  digitalWrite(GLED, 0);

  // 대장 조건 판별 시작
  if (st == 0) {
    digitalWrite(RLED, 1); // 1순위: 누르면 방향 상관없이 무조건 빨간색 점등!
  } 
  else {
    // 버튼을 누르지 않은 틈새 영역(else) 안에서만 2순위 방향 판단 적용
    if (vx < 500) {
      digitalWrite(YLED, 1); // 왼쪽일 때 노란불
    } 
    else if (vx > 3500) {
      digitalWrite(GLED, 1); // 오른쪽일 때 초록불
    }
  }
  
  delay(20); 
}
