1. 오늘 사용할 하드웨어 알아보기

우리가 사용하는 Keyestudio ESP32 PLUS 보드는 기본 아두이노 우노(UNO)보다 연산 속도가 10배 이상 빠르고 메모리가 풍부하여 향후 인공지능(TinyML) 모델을 탑재할 수 있는 고성능 보드입니다.

⚠️ 전압 주의! 조이스틱의 VCC는 반드시 보드의 5V가 아닌 3.3V 전원에 연결해야 전압 포화(Clipping) 현상 없이 부드러운 원형 데이터 수집이 가능합니다.

💡 조이스틱 안에는 뭐가 들어있을까?

조이스틱의 이름에 붙은 VRx, VRy에서 VR은 Variable Resistor(가변저항)의 약자입니다. 조이스틱은 내부적으로 '전기 수도꼭지' 역할을 하는 가변저항 2개가 십자 형태로 겹쳐진 구조입니다. 손잡이를 밀면 저항이 변하고, 그에 따라 변화하는 전압의 크기를 보드가 숫자로 읽어내는 것입니다.

3. [예제 1] 조이스틱 값 시리얼 모니터 확인

조이스틱의 X축, Y축 아날로그 센서 값을 읽어 시리얼 모니터에 출력하는 기본 코드입니다. 아날로그 입력 함수(analogRead)를 쓸 때는 setup() 안의 pinMode 설정을 생략할 수 있어 코드가 매우 간결해집니다.

💡 중요 포인트: 아두이노 우노와 달리 ESP32 보드는 아날로그 입력 해상도가 12비트(0 ~ 4095)입니다. 손잡이를 건드리지 않고 가만히 두었을 때 2000 안팎의 숫자가 출력되면 정상입니다!

int X = 36; // X축 핀 번호
int Y = 39; // Y축 핀 번호

void setup() {
  Serial.begin(115200); // 고속 시리얼 통신 시작 (모니터 속도도 115200 설정!)
}

void loop() {
  int vx = analogRead(X); // X축 값 읽기 (0 ~ 4095)
  int vy = analogRead(Y); // Y축 값 읽기 (0 ~ 4095)

  // 시리얼 모니터에 "X값 , Y값" 형태로 출력
  Serial.print(vx);
  Serial.print(" , ");
  Serial.println(vy);

  delay(100); // 눈으로 확인하기 편하게 0.1초마다 업데이트
}

4. [예제 2] 조이스틱 방향에 따른 신호등 제어

예제 1번 코드의 틀을 그대로 유지하면서, 추가되는 코드를 작성해 봅시다. 조이스틱을 특정 방향으로 확실하게 밀었을 때(값의 경계가 500 미만이거나 3500 이상일 때) 각기 다른 색상의 신호등 LED가 켜집니다.

int X = 36; 
int Y = 39; 

int RLED = 13; // [추가] 빨강 LED 핀
int YLED = 12; // [추가] 노랑 LED 핀
int GLED = 14; // [추가] 초록 LED 핀

void setup() {
  Serial.begin(115200); 

  pinMode(RLED, OUTPUT); // [추가] 출력 모드 설정
  pinMode(YLED, OUTPUT); // [추가] 출력 모드 설정
  pinMode(GLED, OUTPUT); // [추가] 출력 모드 설정
}

void loop() {
  int vx = analogRead(X); 
  int vy = analogRead(Y); 


  // [추가] 매 순간 모든 LED를 끄고 시작 (꺼짐 조건 자동 해결)
  digitalWrite(RLED, 0);
  digitalWrite(YLED, 0);
  digitalWrite(GLED, 0);

  // [추가] 조이스틱 방향 판별 (if - else if 구조 사용)
  if (vx < 500) {
    digitalWrite(YLED, 1); // 왼쪽으로 밀면 노란불 ON
  } 
  else if (vx > 3500) {
    digitalWrite(GLED, 1); // 오른쪽으로 밀면 초록불 ON
  } 
  else if (vy > 3500) {
    digitalWrite(RLED, 1); // 위로 밀면 빨간불 ON
  }

  Serial.print(vx);
  Serial.print(" , ");
  Serial.println(vy);

  delay(20); // [변경] 부드러운 반응을 위해 시간 단축
}

🔥 [최종 도전 과제] 버튼 우선순위 복합 제어 시스템

오늘 배운 디지털 입력, 아날로그 입력, 제어문 알고리즘을 총동원하여 아래의 문제를 해결하는 스케치 코드를 직접 완성해 보세요!

📋 시스템 요구 조건 (시나리오)
  • 조이스틱의 내장 버튼(SW)을 누르면 조이스틱의 방향과 상관없이 무조건 빨간색 LED가 켜집니다.
  • 버튼을 누르지 않았을 때만 조이스틱 방향을 검사합니다.
    • 조이스틱을 왼쪽으로 밀면 노란색 LED가 켜집니다.
    • 조이스틱을 오른쪽으로 밀면 초록색 LED가 켜집니다.
  • 버튼도 누르지 않고, 조이스틱도 움직이지 않는 평상시(그 외의 경우)에는 모든 LED가 꺼집니다.

💡 하드웨어 힌트 (역논리)

조이스틱 버튼(SW)은 INPUT_PULLUP 상태입니다. 따라서 버튼을 클릭하여 누르고 있을 때 디지털 신호값은 0 (LOW)이 되며, 손을 떼고 있을 때 1 (HIGH) 상태가 됩니다.

🧠 알고리즘 핵심 힌트 (우선순위)

컴퓨터는 코드를 위에서부터 아래로 순서대로 검사하며 조건문을 실행합니다. 가장 강력한 조건인 "버튼 누름"을 첫 번째 if 조건으로 배치하고, 조이스틱 판별 코드들은 else if 구조로 묶어야 버튼이 눌렸을 때 아래 조건들을 쳐다보지 않고 건너뛰게 됩니다.
